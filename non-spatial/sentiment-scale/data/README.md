# Data

This project runs on **Reddit Pushshift dumps**: per-subreddit, append-only
archives of comments and submissions, distributed as compressed newline JSON
(one JSON object per line, `.zst` / `.bz2`). They are free, public, and cover
multiple years, which is what makes a sentiment *time series* possible.

## How to obtain a subreddit dump

1. The monthly site-wide and per-subreddit archives are mirrored by the
   `r/pushshift` community and on the Academic Torrents project:
   https://academictorrents.com/ (search "reddit pushshift"). The per-subreddit
   files are far smaller than the site-wide monthly dumps and are the right
   starting point.
2. Download the dump for each subreddit listed in `config/reddit.yaml` and place
   it under `data/raw/`, for example:

   ```
   data/raw/technology_comments.zst
   data/raw/science_comments.zst
   ```

3. The ingest step decompresses the dump, keeps the configured `text_field` and
   the `created_utc` timestamp, filters to the configured date range, and writes
   cleaned Parquet under `data/raw/`. See `USAGE.md` for the full workflow.

A single comment record looks like:

```json
{"body": "this is great news", "created_utc": 1546300800, "subreddit": "technology"}
```

## Alternative source

A large public **Twitter/X academic dataset** works as a drop-in alternative for
short-text sentiment (for example the Sentiment140 corpus of 1.6M labelled
tweets, or an academic-access X archive). The pipeline is unchanged; only the
loader differs, since the per-document record is just `(text, timestamp)`.

Raw dumps and Parquet outputs are git-ignored and regenerated by the ingest
step.
